## Security and Workflow Decisions
- Treat the committed `ADMIN_SECRET` in the public repo as compromised.
- First external gate: rotate `ADMIN_SECRET` in Replit Secrets before or during repo cleanup, then replace the committed literal with secret-managed access only.
- Run a repo-wide hardcoded-secret audit as part of onboarding, not as a later cleanup.
- Do not move off Replit services in this phase.
- Accept Replit database limitations by design:
  - local compile/build/test workflows should work on the Mac
  - DB-backed validation remains a Replit responsibility
- Standard working model:
  1. Replit remains the main runtime and deploy target.
  2. GitHub `main` remains the deployable branch.
  3. Codex and Claude both work from feature branches, never directly on `main`.
  4. One agent/session gets one branch or one worktree.
  5. Merge to GitHub first, then pull/sync in Replit, then validate and deploy from Replit.
- Codex-specific workflow:
  - prefer Codex app worktrees for parallel tasks
  - use the checked-in local environment so each worktree can self-bootstrap
- Claude-specific workflow:
  - run Claude in the repo root or in a dedicated feature branch checkout
  - rely on `CLAUDE.md` plus `.claude/settings.json` for repo behavior
- Shared branch naming default:
  - `codex/<task-name>` for Codex-created branches
  - `claude/<task-name>` for Claude-created branches
